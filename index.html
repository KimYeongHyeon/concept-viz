The Problem

왜 새로운 검토 도구가 필요한가

AI 매핑의 정확도는 향상되었지만, 의료진이 결과를 검증하는 단계에서 여전히 병목이 발생합니다. 기존 OHDSI 도구들은 단일 concept 조회와 코호트 생성에 특화되어 있어, "AI 후보 검토"라는 중간 워크플로우가 빠져있습니다.

⚠ 현재 (Athena + Usagi + ATLAS 파편화)
  • ✕Athena에서 concept를 하나씩 검색·조회 (한 번에 하나만)
  • ✕별도 스프레드시트에 결과 복사해서 수동 비교
  • ✕계층 구조 탐색이 링크 클릭 중심, 맥락 상실
  • ✕실제 환자 수·분포 확인 불가 (vocabulary만)
  • ✕criteria 원문과 매핑 결과가 분리됨
  • ✕매핑이 끝나야 feasibility 확인 가능 → 재작업 반복
✓ concept-viz (통합 검토 워크플로우)
  • →DOI Tree로 후보들을 동시 비교, 맥락 유지
  • →드래그·래소 선택으로 concept set 시각적 구성
  • →선택할 때마다 cohort 크기 실시간 갱신
  • →CDM 환자 수·분포를 concept 옆에 오버레이
  • →criteria 원문 · AI 추론 근거 · 매핑 결과를 한 화면에
  • →Anchor 기반 sibling 전개로 빠진 concept 발견
Our Approach

세 가지 시각화 기둥으로 검토 효율을 높입니다
