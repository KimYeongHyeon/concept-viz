● ATLAS Positioning

ATLAS와 어디서 겹치고,
무엇이 시점에서 다른가

OHDSI ATLAS는 OHDSI 생태계의 표준 도구이며, 본 프로젝트가 다루는 일부 시각화 영역(계층 트리·attrition·환자 수)을 이미 제공합니다. 이 문서는 겹침을 정직하게 정리하면서, 우리의 핵심 차별점이 "피드백이 일어나는 시점"임을 보여줍니다.

섹션 1핵심은 시간 — 매핑 결정 *후* vs *동안*

ATLAS의 모든 시각화 피드백 (cohort 크기, attrition, 특성화)은 "매핑이 끝난 후 cohort generation을 실행해야" 나타납니다. 이 generation은 보통 수 분에서 수 시간이 걸립니다 (CDM 크기와 쿼리 복잡도에 따라).

concept-viz는 같은 정보를 "매핑 결정이 일어나는 그 순간" 보여줍니다. 의료진이 concept를 추가·교체하면 환자 수가 즉시 갱신됩니다 (수십~수백 ms). 이 차이는 단순한 속도 향상이 아니라 "의사결정의 종류"를 바꿉니다.

ATLAS는 cohort generation 후에야 첫 피드백 — 한 사이클 5–30분 / concept-viz는 모든 인터랙션마다 피드백 — 한 사이클 1초 미만

한 매핑 결정에 평균 50번의 "이걸 바꾸면?"가 발생한다고 가정

의료진은 한 criteria의 매핑을 결정할 때 보통 수십 번의 가설 ("이 concept를 빼면 환자 수가 어떻게 될까?", "이 형제를 추가하면?")을 검토합니다.

⚠ ATLAS 사이클
~12 시간
50회 × 평균 14분 (concept set 수정 + cohort 재생성 대기) — 현실적으로 의료진은 이 비용 때문에 대부분 첫 매핑을 그대로 받아들임
✓ concept-viz 사이클
~1 분
50회 × ~1초 (UI 인터랙션 + 즉시 갱신) — 같은 시간에 700배 더 많은 가설 검증 가능
핵심: 단순 속도 향상이 아니라 "의사결정의 종류"가 바뀝니다. ATLAS는 "최선이 무엇인지 한 번 정해 받아들이는" 결정을, concept-viz는 "여러 시나리오를 비교하며 더 나은 것에 수렴하는" 결정을 가능하게 합니다.

섹션 3정직한 포지셔닝 — 강한 카드와 약한 카드

"ATLAS도 한다"는 reviewer 코멘트에 대비해 미리 입장을 명확히 합니다. 아래 강점은 단독 publication 가능 수준, 약점은 정량적 증거(사용자 연구 등)로 보강 필요한 영역입니다.

✓ 강한 카드 (확실한 차별점)
  • AI 매핑 추천 검토 인터페이스 — ATLAS 0% 겹침. AI 매핑이 등장한 후 의료진의 검증 워크플로우가 비어 있음을 메움
  • 실시간 feasibility 피드백 — ATLAS도 attrition을 보여주지만 cohort generation 후. 우리는 selection 즉시 → 의사결정 사이클 700배 단축 (섹션 1 참조)
  • Anchor 기반 gap 감지 — ATLAS 0%. AI가 놓친 sibling concept를 시각적으로 자동 노출
  • Criteria 원문 컨텍스트 유지 — ATLAS는 cohort definition과 protocol 원문이 분리. 우리는 같은 화면에서 매핑 판단의 근거 유지
⚠ 약한 카드 (인정 + 보강 필요)
  • Concept hierarchy 시각화 — ATLAS도 트리 제공. DOI Tree + Semantic Zoom의 우월성은 사용자 연구로 증명 필요
  • Concept set 구성 UX — 동일 목표. 드래그/래소 효율성을 정량적으로 (선택 시간, 오류율) 비교해야 설득력
  • Vocabulary overview — ATLAS Achilles도 도메인 분포 차트 제공. Treemap의 인터랙티브성이 차이지만, 사용 빈도 자체가 낮을 수 있음
  • Production 통합 — ATLAS만 WebAPI 직결. 우리는 매핑 결과를 export해야 함. "별도 도구 → ATLAS로 옮김" 마찰 인정
한 줄 포지셔닝 (논문·grant 응답용):
concept-viz는 ATLAS를 대체하지 않는다. ATLAS의 강점인 cohort characterization, pathway, production cohort generation은 그대로 활용하면서, ATLAS에 없는 "AI 추천 매핑을 의료진이 검토하는 단계"를 메운다. 매핑이 결정된 *후* 통계를 보는 ATLAS와, 매핑이 결정되는 *동안* 실시간 피드백을 주는 concept-viz는 시점이 다르다.

섹션 4시점이 다르다는 것이 의미하는 것

의료진의 인지 부담 변화

ATLAS 워크플로우에서 의료진은 "매핑 → 기다림 → 결과 → 무엇을 바꿀지 기억해서 → 다시 매핑" 사이클을 돕니다. 각 사이클 사이에 인지 컨텍스트 손실(왜 이 concept를 추가했는지, 어떤 가설을 검증 중이었는지)이 발생합니다. concept-viz는 가설과 검증이 같은 인지 세션 안에서 일어납니다.

탐색 가능 공간의 차이

ATLAS에서 50회의 "if-then" 검증은 비현실적이라 의료진은 보통 처음 떠올린 5~10개 변형만 시도합니다. concept-viz는 같은 시간에 100~500개 시나리오 탐색이 가능. 이는 최선의 매핑에 도달할 확률을 직접적으로 높입니다.

"AI 의존" 패턴 변화

검증 비용이 높으면 (ATLAS) 의료진은 AI 추천을 그대로 받아들이는 압박을 받습니다. 검증 비용이 거의 0에 가까우면 (concept-viz) 의료진은 적극적으로 도전·수정합니다. 같은 AI 모델을 쓰더라도 검증 도구가 final mapping 품질을 좌우합니다.

재현성·이력 관리

ATLAS는 free-text 메모로 매핑 근거를 기록하지만 의료진이 일관되게 작성하기 어렵습니다. concept-viz는 인터랙션 자체가 로그됩니다 (어떤 후보를 보고, 어떤 것을 거부했고, 왜 변경했는지). 후속 연구자가 동일 매핑을 재현하기 쉬워집니다.
